pelayout is a script in CIMEROOT/scripts/Tools.

$ ./pelayout --help
usage: pelayout [-h] [-d] [-v] [-s] [-format FORMAT] [-set-nasks SET_NTASKS]
                [-set-nthrds SET_NTHRDS] [-header HEADER] [-no-header]
                [-caseroot CASEROOT]

Display and/or change the case's PE layout.

optional arguments:
  -h, --help            show this help message and exit
  -d, --debug           Print debug information (very verbose) to file /Users/
                        aliceb/cesm2_0_alpha07/cime/scripts/Tools/pelayout.log
  -v, --verbose         Add additional context (time and file) to log messages
  -s, --silent          Print only warnings and error messages
  -format FORMAT, --format FORMAT
                        Format the PE layout items for each component (see
                        below)
  -set-nasks SET_NTASKS, --set-ntasks SET_NTASKS
                        Total number of tasks to set for the case
  -set-nthrds SET_NTHRDS, --set-nthrds SET_NTHRDS, -set-nthreads SET_NTHRDS, --set-nthreads SET_NTHRDS
                        Number of threads to set for all components
  -header HEADER, --header HEADER
                        Custom header for PE layout display
  -no-header, --no-header
                        Do not print any PE layout header
  -caseroot CASEROOT, --caseroot CASEROOT
                        Case directory to reference

This utility allows the CIME user to view and modify a case's PE layout.

Interpreted format sequences are:"
%%  a literal %
%C  the component name
%T  the task count for the component
%H  the thread count for the component
%R  the PE root for the component

Standard format extensions, such as a field length and padding are supported.
Example:
  For NTASKS_ATM=480, NTHREADS_ATM=2 and ROOTPE_ATM, "%C: %06T/%+H" will show
ATM: 000480/+2
Note that since %R was omitted, the root PE was not shown.
Python dictionary-format strings are also supported. For instance,
--format "{C:4}", will print the component name padded to 4 spaces.
